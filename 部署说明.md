## 方式一：通过命令行部署（推荐）

### 步骤 1：完成 Vercel 登录

1. **打开浏览器访问以下链接**（如果之前的链接已过期，运行 `vercel login` 获取新链接）：
   ```
   https://vercel.com/oauth/device?user_code=NXVJ-MHXW
   ```
   或者重新运行：
   ```powershell
   vercel login
   ```

2. **在浏览器中完成登录**：
   - 使用 GitHub、GitLab、Bitbucket 或邮箱登录
   - 授权 Vercel 访问

3. **返回终端**，登录会自动完成

### 步骤 2：执行部署

登录完成后，运行以下命令：

```powershell
# 部署到生产环境
vercel --prod --yes
```

或者使用部署脚本：

```powershell
.\deploy.ps1
```

### 步骤 3：获取访问链接

部署成功后，Vercel 会显示：
- **生产环境 URL**：`https://your-project.vercel.app`
- **预览 URL**：`https://your-project-xxx.vercel.app`

将这个 URL 分享给他人即可访问！

## 方式二：通过 Vercel 网站部署（更简单）

如果命令行登录有问题，可以使用网站部署：

### 步骤 1：准备代码仓库

1. **将代码推送到 GitHub**（如果还没有）：
   ```powershell
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/learn_learn.git
   git push -u origin main
   ```

### 步骤 2：在 Vercel 网站部署

1. **访问** https://vercel.com
2. **使用 GitHub 账号登录**
3. **点击 "Add New..." → "Project"**
4. **导入你的仓库** `learn_learn`
5. **Vercel 会自动检测配置**（已配置好 `vercel.json`）
6. **点击 "Deploy"**
7. **等待部署完成**（约 1-2 分钟）
8. **获取访问链接**

## 常见问题

### Q: 登录链接过期了怎么办？
A: 重新运行 `vercel login` 获取新链接。

### Q: 部署失败怎么办？
A: 
1. 检查 `dist` 目录是否存在
2. 运行 `npm run build` 确保构建成功
3. 查看错误日志：`vercel logs`

### Q: 如何绑定自定义域名？
A: 
1. 在 Vercel 项目设置中添加域名
2. 按照提示配置 DNS 记录

### Q: 如何设置环境变量？
A: 
1. 在 Vercel 项目设置 → Environment Variables
2. 添加需要的环境变量（如 API Key）
